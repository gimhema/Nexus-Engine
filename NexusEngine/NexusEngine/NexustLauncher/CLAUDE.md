# NexusLauncher — CLAUDE.md

이 파일은 Claude Code가 NexusLauncher 프로젝트에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

NexusLauncher는 Nexus MMORPG의 로컬 서버 호스팅 및 관리를 위한 데스크탑 런처입니다.

### 배경 및 방향성

- 게임은 **멀티플레이 기반 단일 구조**로 설계됨 — 싱글/멀티 코드 분기 없음
- 싱글플레이 시에는 유저가 **로컬 서버를 직접 호스팅**하는 방식
- 멀티플레이(친구 초대) 시에는 호스트가 IP를 공유하면 됨 — 확장이 자연스러움
- 서버는 런처가 백그라운드로 실행 관리. 유저는 런처만 실행하면 됨

### 런처 실행 흐름

```
런처 실행
  └─ 서버 프로세스 백그라운드 spawn
       └─ 서버 준비 확인 (포트 응답 폴링)
            └─ UE5 클라이언트 실행 (localhost 접속)
```

### 클라이언트와의 역할 분리

- **런처**: 서버 생명주기 관리, 호스트 권한 기능, 서버 모니터링
- **UE5 클라이언트**: 서버 주소를 받아 접속만 담당 — 서버 관리 코드 없음
- 이 분리 덕분에 클라이언트 코드가 단순해지고, 멀티/싱글 모드 분기가 불필요

## 기술 스택

| 항목 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | **Tauri 2** | MIT/Apache 2.0, OS 기본 WebView 사용으로 경량 |
| 백엔드 | **Rust** | 단일 바이너리, 낮은 메모리, 프로세스 관리에 적합 |
| UI | **HTML / CSS / TypeScript** | 디자인 자유도 높음, 핫리로드, 개발자도구 디버깅 |
| 라이선스 | MIT / Apache 2.0 | 상용 배포 제한 없음 |

### Electron을 선택하지 않은 이유

런처는 유저가 닫지 않고 상주시키는 경우가 많음.
Electron은 Chromium 번들로 100MB+, 상주 메모리도 높음.
Tauri는 OS 기본 WebView를 사용해 수MB 설치 크기, 20~30MB 수준 메모리.

## 런처 핵심 기능

### 1. 서버 생명주기 관리
- 서버 프로세스 시작 / 종료
- 크래시 감지 및 자동 재시작 (옵션)
- 서버 준비 상태 확인 (TCP 포트 폴링)

### 2. 서버 모니터링
- 현재 접속자 수 표시
- 서버 로그 스트리밍
- 서버 가동 시간

### 3. 호스트 권한 기능 (멀티플레이)
- 접속자 목록 조회
- 특정 유저 킥
- IP/포트 설정 변경

### 4. 클라이언트 실행
- 서버 준비 완료 후 UE5 클라이언트 자동 실행
- 접속 주소(localhost 또는 외부 IP) 클라이언트에 전달

## 서버와의 통신 방식

런처가 서버 상태를 조회하고 관리 명령을 전달하는 방법:

- **옵션 A**: 서버 내 경량 HTTP 관리 API 추가 (REST) — 런처가 HTTP로 폴링/명령
- **옵션 B**: 별도 관리 소켓 (TCP/Unix socket) — 런처 전용 채널
- **옵션 C**: 서버 stdout 파싱 + stdin 명령 — 간단하지만 제한적

> **미결정**: 서버(NexusEngine) 측에 관리 API를 어떤 방식으로 추가할지 확정 필요.
> 현재 유력 후보는 옵션 A (경량 HTTP REST API).

## 프로젝트 구조 (예정)

```
NexustLauncher/
├── src-tauri/          # Rust 백엔드
│   ├── src/
│   │   ├── main.rs         # Tauri 앱 진입점
│   │   ├── server.rs       # 서버 프로세스 관리
│   │   ├── monitor.rs      # 서버 상태 모니터링
│   │   └── commands.rs     # Tauri 커맨드 (프론트↔백엔드 브릿지)
│   └── Cargo.toml
├── src/                # 프론트엔드 (HTML/CSS/TS)
│   ├── main.ts
│   ├── pages/
│   │   ├── home.ts         # 메인 화면 (서버 상태, 시작/종료)
│   │   ├── players.ts      # 접속자 관리
│   │   └── settings.ts     # 서버 설정
│   └── styles/
├── package.json
└── tauri.conf.json
```

## 서버 연동 참고

서버(NexusEngine) 관련 상수:

| 항목 | 값 |
|---|---|
| TCP 게임 포트 | 7070 |
| UDP 게임 포트 | 7071 |
| 관리 API 포트 | 미정 (7072 예정) |
| 서버 실행 파일 | `NexusEngine` (Linux) / `NexusEngine.exe` (Windows) |

## 개발 규칙

- 소스 코드 주석은 한국어로 작성
- Rust 백엔드와 TS 프론트엔드 간 인터페이스는 `commands.rs`에 집중
- 서버 관련 하드코딩 값은 `tauri.conf.json` 또는 별도 설정 파일로 분리
